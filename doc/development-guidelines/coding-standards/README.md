# 代码风格规范

此目录用于存储代码风格规范、命名约定等编码标准文档。

## 前端代码规范

### Vue 组件命名
- 组件名称应使用 PascalCase 命名法
- 文件名应与组件名称保持一致
- 基础组件应使用 Base 前缀
- 业务组件应使用功能相关的名称

### JavaScript/TypeScript 规范
- 使用 ES6+ 语法
- 变量声明使用 const/let，避免使用 var
- 函数使用箭头函数或函数声明
- 代码缩进使用 2 个空格
- 字符串使用单引号
- 行尾不使用分号

### CSS 规范
- 使用 BEM 命名约定
- 避免使用 ID 选择器
- 优先使用类选择器
- 样式应模块化，避免全局污染

## 后端代码规范

### Node.js/Express 规范
- 目录结构清晰，按功能模块组织
- 路由定义清晰，遵循 RESTful API 设计原则
- 中间件使用合理，职责单一
- 错误处理统一，使用 try/catch 或错误中间件

### 命名约定
- 变量名使用 camelCase
- 函数名使用 camelCase
- 类名使用 PascalCase
- 常量使用 UPPER_CASE
- 文件和目录名使用 kebab-case

## 代码审查要求

- 代码提交前应进行自我审查
- 提交信息应清晰描述变更内容
- 代码审查应关注代码质量、性能和安全性
- 审查通过后才能合并到主分支
