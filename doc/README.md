# 项目文档目录

此目录用于存放项目相关的各类文档，按照功能和类型进行分类管理。

## 目录结构

```
doc/
├── system-definition/         # 系统定义相关文档
│   ├── architecture/         # 架构图、技术选型说明
│   └── requirements/         # 需求规格说明书、用户故事
└── development-guidelines/   # 开发规范相关文档
    ├── coding-standards/     # 代码风格规范、命名约定
    ├── workflow/             # 开发流程、版本控制规范
    └── best-practices/       # 最佳实践、常见问题解决方案
```

## 文档管理规范

- 所有文档应使用Markdown格式编写
- 文件名应使用小写字母，单词之间用连字符(-)连接
- 文档应保持更新，反映项目的最新状态
- 重要文档应包含版本信息和更新记录
